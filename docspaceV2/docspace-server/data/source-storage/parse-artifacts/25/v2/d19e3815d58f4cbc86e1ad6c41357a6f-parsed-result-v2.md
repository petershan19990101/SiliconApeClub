# 2.2.1 健康探测

- ![Page 1 Image 1](/api/documents/25/parse-artifacts/316/content)

1. 机构增加状态字段（0-正常状态 1-熔断状态）

1. 机构初始化新增时状态为“正常状态”；健康探测任务，定时进行检测；若指标探测异常则转为“熔断状态”，正常则不做状态转换。具体规则

1. 若探测无订单数据，则不进行健康状态标记处理。

1. 指标探测，校验抽查单位时间内是否达到异常交易指标，达到则熔断，未达到则正常

1. 恢复机制，熔断状态机构，需检测是否满足重试探测机制，满足则进行重试探测，检查恢复指标，符合条件则恢复正常状态，不符合则不进行恢复。

1. 指标探测规则如下表：

| 探测类型 | 时间指标 | 数量指标 | 处理方式 |
| --- | --- | --- | --- |
| 服务异常 | 5mine内服务异常率>80% | 或连续异常50笔及以上 | 熔断+短信告知 |
| 服务超时 | 10mine内服务超时120s>80% | 或连续超时100笔及以上 | 熔断+短信告知 |
| 审批拒绝 | 10mine内服务审批拒绝率>70% | 或连续审批拒绝100笔及以上 | 熔断+短信告知 |

1. 短信内容：

- 系统监测【机构简称】，【告警指标内容：例：5mine内服务异常率>80%/服务连续异常50笔以上】，已触发熔断机制，请联系合作方。

- 系统监测【机构简称】，已解除熔断，恢复正常。

# 2.2.2 恢复流程

![Page 1 Image 3](/api/documents/25/parse-artifacts/318/content)

1. 熔断状态机构，可通过恢复任务定时监测灰度测试的交易订单状态，满足恢复要求则更改机构状态。

1. 熔断状态重试机制，间隔周期内不做重试处理，重试间隔时间内可以发起重试单子，第一个间隔周期为熔断标记开始时间。后续每次重试间隔时间后为间隔周期。

| 间隔周期 | 重试间隔时间 | 重试单（Max） | 重试次数 | 熔断恢复标准 |
| --- | --- | --- | --- | --- |
| 30mine | 5mine | 5 单 | 20次 | 服务审批成功数量>3单 且 审批时长小于120s |
|  |  |  |  |  |

- 3、若重试次数到达后一直未恢复，则不在进行恢复任务，后续进行人工状态调整。

# 2.2.3 资金路由

- ![Page 1 Image 4](/api/documents/25/parse-artifacts/319/content)

1. 资金方路由增加，机构校验逻辑，校验机构状态是否正常，正常则继续进行路由优先流程，状态为熔断状态，则校验是否满足重试机制，满足则直接输出该资金方；不满足重试机制则直接剔除路由列表。

1. 判断是否满足重试机制，及判断该机构熔断时间是否在重试周期内，以及重试周期内是否达到重试单数量。

1. 产品维度支持是否支持二次路由设置，在产品维度可设置二次路由开关，以及二次路由订单超时时间，具体规则如下：

- A、订单标记熔断标识后，退回消费队列

- B、二次消费时，校验资产是否支持二次路由，订单是否超时，条件满足：二次路由，订单时间未超时，重新筛选资金方

- C、存在新资金方则进行二次分发，不存在则按照拒绝处理，返回订单进行兜底策略判断，进行兜底放款或者拒绝。

- D、二次路由单校验，二次路由单子包含熔断产生单子以及正常审批拒绝单子，需二次路由时校验首次资金方机构标识，二次路由时剔除该机构，进行筛选。
